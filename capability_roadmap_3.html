C1 – C6
C1진행 중

AI 에이전트 코딩 & 프롬프트 엔지니어링

AI로 코드를 만들고, 결과물의 한계와 적용지점을 스스로 판단한다.

지금 어디

Claude Code를 세팅했고, AI를 학습 파트너로 매일 쓰는 중. Day 5엔 AI의 부정확한 설명을 직접 검증으로 바로잡았고, Day 7엔 OOP로 코드를 객체 단위로 구조화(early return·if __name__ 등 관례 체득).

다음 한 걸음

실전 기능에 AI를 적용하며 “AI가 잘한 곳 vs 못한 곳”을 기록 → Phase 4에서 깊어짐.

C2진행 중

데이터 구조 기본기

list · dict · tuple · set 등 자료구조를 이해하고 적재적소에 쓴다.

지금 어디

Python 멘토십 Day 7 완주(Week 1)! 자료구조 4종(list·dict·tuple·set)+파일 I/O에 더해, 클래스로 '나만의 자료구조(Account)'를 직접 설계.

다음 한 걸음

상속·다형성(OOP 2)으로 자료구조 설계를 심화 → 이후 프로젝트에서 강화.

C3시작 전

PostgreSQL & MongoDB / 정규화 · 반정규화

관계형·문서형 DB를 다루고, 정규화/반정규화 트레이드오프를 판단해 선택한다.

지금 어디

PostgreSQL은 현대화의 목표 스택. 실무에서 여러 테이블을 다뤄온 경험이 있어 정규화 개념은 빨리 와닿을 토대.

다음 한 걸음

같은 데이터를 PostgreSQL(정규화)과 MongoDB(반정규화)로 각각 설계하며 대비 → Phase 3.

C4진행 중

Git 협업 & 머지 전략

브랜치·PR로 협업하고, rebase 머지와 squash 머지의 장단점을 안다.

지금 어디

첫 Pull Request 완성! 브랜치→push→PR→리뷰(Files changed)→병합→pull 전체 사이클을 직접 경험. git rm으로 불필요 파일 정리까지. (Day 6 충돌 해결 + Day 7 첫 PR)

다음 한 걸음

rebase / squash 머지 + 코드 리뷰 주고받기 → Phase 2 완성.

C5진행 중

고객 중심의 유연한 의사결정

기술보다 사용자 가치를 기준으로 합리적이고 유연하게 결정한다.

지금 어디

500명 규모 조직을 위한 현대화 프로젝트 자체가 무대. 실무 경험이 그대로 자산이 되는 영역.

다음 한 걸음

실무 감각을 “증명 가능한 의사결정 사례”로 전환 → Phase 5.

C6시작 전

백로그 & 우선순위 관리

할 일을 백로그로 정리하고, 우선순위를 세워 굴린다.

지금 어디

현대화 프로젝트(170여 개 테이블·문서 라이브러리)가 백로그 관리를 자연히 요구하는 상황.

다음 한 걸음

프로젝트용 우선순위 백로그 보드를 실제로 한 번 운영 → Phase 5.
